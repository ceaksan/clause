# Clause - Architecture

Minimal, ephemeral note-taking companion for Claude Code sessions: a floating SwiftUI window paired with an MCP server, communicating over a Unix domain socket.

**Key responsibilities per module:**

- `ClauseShared`: Single source of truth for wire types. Both processes import this; no duplication.
- `ClauseApp/Server/SocketServer.swift`: Accepts POSIX connections, deserializes IPCRequest, dispatches to NoteStore on MainActor, serializes IPCResponse.
- `ClauseApp/Store/NoteStore.swift`: Central state; all mutations go through here; debounce timer triggers JSON snapshot.
- `ClauseMCP/main.swift`: Registers 6 MCP tools with swift-sdk; translates tool calls to IPCRequest; sends over NWConnection to app socket; returns IPCResponse as MCP result.

## Data Flow

Two independent flows; both converge on NoteStore.

**Flow 1 - Claude to App (MCP tool call):**

```
Claude Code
  --stdin/JSON-RPC-->
    clause-mcp (ClauseMCP binary)
      --> NWConnection (Network.framework, client)
        --Unix socket (~/.clause/clause.sock)-->
          SocketServer (POSIX, server, ClauseApp)
            --> handleRequest() dispatched to @MainActor
              --> NoteStore.addNote / editNote / deleteNote / etc.
                --> IPCResponse serialized
              <-- IPCResponse
            <--Unix socket-->
          NWConnection receives response
        --> stdout / JSON-RPC result
      --> Claude Code receives tool result
```

**Flow 2 - User to App (direct UI interaction):**

```
User types in InputBar
  --> submitNote() called
    --> NoteStore.addNote(source: .user)
      --> @Observable triggers SwiftUI diffing
        --> NoteRowView rendered in ContentView list
          --> debounce timer fires after idle period
            --> JSON snapshot written to ~/.clause/sessions/{id}.json
```

## MCP Tools

The ClauseMCP binary exposes 6 tools over the MCP stdio protocol. All tools communicate to ClauseApp via Unix socket.

```
set_session(id, directory)          # Initialize session context; shows floating window
add_note(text, type?)               # Add a note, todo, or warning (default: note)
list_notes(type?, source?)          # List notes with optional type/source filters
edit_note(id, text?, type?, completed?)  # Update one or more fields of an existing note
delete_note(id)                     # Remove a single note by UUID
clear_notes()                       # Remove all notes in the current session
```

**Tool parameter details:**

| Tool | Key Parameters | Returns |
|------|---------------|---------|
| set_session | id: String, directory: String | IPCResponse (ok or error) |
| add_note | text: String (max 4096), type: note/todo/warning | Created Note |
| list_notes | type?: note/todo/warning, source?: claude/user | [Note] array |
| edit_note | id: UUID, text?, type?, completed?: Bool | Updated Note |
| delete_note | id: UUID | ok or error |
| clear_notes | none | ok or error |

## Data Model

**Note** (primary entity, in-memory + persisted in session JSON):

| Field | Type | Notes |
|-------|------|-------|
| id | UUID | Primary key, auto-generated on creation |
| timestamp | Date | ISO 8601, set at creation |
| source | enum: claude, user | Indicates origin of the note |
| type | enum: note, todo, warning | Display badge and filtering |
| text | String | Max 4096 chars; silently truncated on input |
| completed | Bool | Default false; relevant for todo type |

**Session** (per-session metadata, serialized alongside notes):

| Field | Type | Notes |
|-------|------|-------|
| id | String | Provided by Claude via set_session |
| directory | String | Working directory of the Claude Code session |
| protocolVersion | String | Always "1" for current implementation |
| pid | Int32 | PID of the clause-mcp process |
| startedAt | Date | Session start timestamp |

Only one session is active at a time. A new set_session call replaces the previous session context.

## Security

- Unix socket directory (~/.clause/) is created with 0o700 (owner read/write/execute only)
- Socket file is accessible only to the current OS user; no network binding
- Single-user-per-home-directory scope; no multi-user concerns
- No inbound network connections; ClauseApp is entirely local
- Note text silently truncated at 4096 chars on ingestion; no buffer overrun risk
- Protocol version checked on every IPC connection; mismatch returns ErrorCode -5
- No sandboxing entitlements required; no capability escalation

## Constraints & Trade-offs

| Decision | Reason | Trade-off |
|----------|--------|-----------|
| POSIX sockets for server binding | NWListener (Network.framework) does not support AF_UNIX server socket creation on macOS | Mixed API: POSIX on server side, NWConnection on client side; higher complexity in SocketServer.swift |
| In-memory state with debounced JSON snapshot | Simple, fast, avoids SQLite dependency | Up to 10 seconds of data loss on crash; acceptable for ephemeral scratchpad use |
| Single active session (new set_session replaces previous) | Avoids multi-session complexity and window management overhead | Parallel Claude Code sessions cannot each maintain independent note lists |
| No encryption on Unix socket | Local-only, same-user IPC; OS-level file permissions are sufficient | Not suitable if multi-user or remote access scenarios were ever required |
| LSUIElement app (no dock icon) | Floating companion; should not appear as a primary app | Requires explicit window activation; no app switcher presence |
| xcodegen for project file | Keeps Clause.xcodeproj out of version control; project structure defined in plain YAML | Requires xcodegen installed; adds one build step before opening Xcode |

## Known Tech Debt

**High priority:**
- [ ] FloatingPanel.swift exists but is not used as the window class; configureWindow() patches a standard NSWindow instead of instantiating NSPanel subclass
- [ ] No accessibility permission handling (required for global hotkey in Milestone 2)

**Medium priority:**
- [ ] Hotkey support not implemented (KeyboardShortcuts dependency added to plan but not yet integrated)
- [ ] Session cleanup not implemented; old ~/.clause/sessions/*.json files accumulate indefinitely (Milestone 2)

**Low priority:**
- [ ] list_notes returns all notes regardless of session if called before set_session
- [ ] No graceful socket cleanup on abnormal app termination; stale .sock file can block next launch

## Code Hotspots

| File | Risk | Reason |
|------|------|--------|
| ClauseApp/Server/SocketServer.swift | High | All IPC routing logic; POSIX + actor boundary crossing; most likely to change with new MCP tools |
| ClauseMCP/main.swift | High | MCP protocol integration + socket client; changes whenever tool signatures evolve |
| ClauseApp/Store/NoteStore.swift | Medium | Central state; all CRUD + persistence; any data model change touches this file |

### Dependency Graph

ClauseApp and ClauseMCP both depend on ClauseShared (static framework). No circular dependencies.

```
ClauseShared (static framework)
  ^               ^
  |               |
ClauseApp     ClauseMCP
(macOS app)   (CLI / MCP stdio server)
```

ClauseApp and ClauseMCP have no compile-time dependency on each other. Runtime coupling is exclusively through the Unix domain socket using the types defined in ClauseShared.
